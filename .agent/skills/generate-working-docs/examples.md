# 使用例

このドキュメントでは、開発作業ドキュメント生成スキルの使用例を記載します。

---

## 例1: クエリ実行機能の開発

### ユーザーの依頼

```
新規開発のドキュメント作ってください
```

### Claudeの応答

```
要件名を教えてください。
英語のケバブケース（例：query-execution, export-csv）を推奨します。
```

### ユーザーの回答

```
query-execution
```

### 生成結果

```
docs/working/20251229_query-execution/
├── requirements.md   # クエリ実行の要件一覧
├── design.md         # クエリ実行ロジックの設計
├── tasklist.md       # 実装タスク一覧
└── testing.md        # テスト手順
```

## ドキュメント完成後のワークフロー

### 1. 各ファイルを編集

生成されたテンプレートを元に、具体的な内容を記載します。

### 2. タスクを進行

`tasklist.md` のチェックボックスを更新しながら開発を進めます。

### 3. テストを実施

`testing.md` の手順に従ってテストを実施し、結果を記録します。

### 4. 開発完了後

1. **永続化ドキュメントを更新**
   - `docs/01_product_requirements.md` など、関連するドキュメントを更新
   - `docs/features/` の機能詳細仕様を更新（必要に応じて）

2. **開発作業ドキュメントをアーカイブ**
   - `docs/archive/` に移動、または削除

3. **重要な設計判断は永続化**
   - 設計書に記載した重要な決定事項は永続化ドキュメントに反映
