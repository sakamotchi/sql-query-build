# テスト手順書 - db-connection-timeout

## 概要

DBが起動していない状態でクエリビルダー・SQLエディタを開いたときに、タイムアウト後にエラーが表示されることを確認します。

## 前提条件

- `npm run tauri:dev` でアプリが起動していること
- 接続設定が保存されていること（PostgreSQLまたはMySQL）

## 手動テスト

### ケース 1: DBが起動していない状態でクエリビルダーを開く

**手順:**

1. DBサーバー（PostgreSQL/MySQL）を停止する
2. アプリを起動する（`npm run tauri:dev`）
3. 接続を選択してクエリビルダー画面を開く
4. サイドバーのデータベースツリーを確認する

**期待結果:**

- ローディングスピナーが表示される
- 30〜35秒以内にスピナーが消え、エラーメッセージが表示される
- 「データベース接続がタイムアウトしました（30秒）」のようなメッセージが見える

**確認結果:**

- [ ] OK / NG

---

### ケース 2: DBが起動していない状態でSQLエディタを開く

**手順:**

1. DBサーバーを停止する
2. アプリを起動する
3. 接続を選択してSQLエディタ画面を開く
4. サイドバーとローディング状態を確認する

**期待結果:**

- ローディングスピナーが表示される
- 30〜35秒以内にスピナーが消え、エラーが表示される

**確認結果:**

- [ ] OK / NG

---

### ケース 3: DB起動後に正常に動作する（回帰テスト）

**手順:**

1. DBサーバーを起動する
2. アプリを起動する
3. 接続を選択してクエリビルダー画面を開く
4. データベースツリーが正常に表示されることを確認する

**期待結果:**

- テーブル一覧が正常に表示される
- タイムアウトエラーが出ない

**確認結果:**

- [ ] OK / NG
