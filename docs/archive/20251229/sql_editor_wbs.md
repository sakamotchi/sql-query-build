## 開発標準・ルール

### Git ワークフロー

- **ブランチ戦略**: Git Flow
  - `main`: 本番リリース用
  - `develop`: 開発統合用
  - `feature/*`: 機能開発用
  - `release/*`: リリース準備用
  - `hotfix/*`: 緊急修正用

### コーディング規約

- **Rust**: Rust標準スタイル (rustfmt)
- **TypeScript**: ESLint + Prettier
- **Vue.js**: Vue公式スタイルガイド
- **コミットメッセージ**: Conventional Commits

### レビュープロセス

- 全てのコードは最低1名のレビュー必須
- クリティカルな機能は2名以上のレビュー
- レビュー後24時間以内にマージ

### テスト基準

- 新機能は必ずユニットテスト作成
- バグ修正は再発防止テスト作成
- E2Eテストは主要フロー全てカバー

## 前提条件・制約事項

### 前提条件

- 開発チームのスキルセット(Rust, Vue.js, Tauri)が十分であること
- 開発環境(PC, ライセンス等)が準備されていること
- テスト用のデータベース環境が利用可能であること
- ステークホルダーからのタイムリーなフィードバックが得られること

### 制約事項

- 対応データベースは PostgreSQL, MySQL, SQLite のみ
- オフラインで動作するデスクトップアプリケーション
- 初期リリースは日本語のみ対応
- クラウド同期機能は次バージョン以降

### 除外事項 (v1.0対象外)

- データベースマイグレーション機能
- ビジュアルER図自動生成
- AIによるクエリ最適化提案
- チーム機能・権限管理
- クエリのバージョン管理
- データベーススキーマ比較
